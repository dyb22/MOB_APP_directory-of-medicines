# Онлайн-справочник лекарств

Мобильное Android-приложение — онлайн-справочник лекарств с поиском, закладками, историей поиска и профилем юзера.

---

## Стек технологий

- **Язык:** Kotlin  
- **Архитектура:** (Clean Architecture): `app`, `data`, `domain`
- **Архитектурные компоненты:** ViewModel, Lifecycle, Fragment KTX

---

## Как запустить приложение

1. Открыть проект в **Android Studio**.
2. Подключить устройство с Android 7.0+ (API 24) или запустить эмулятор.
4. Выполнить запуск: **Run → Run 'app'** (или зелёная кнопка Run).  
   Либо из терминала в корне проекта:
   ```bash
   ./gradlew installDebug
   ```
5. На устройстве/эмуляторе откроется приложение «Онлайн-справочник лекарств».

---

## Где находятся моки

Мок-данные используются временно вместо реального API и локального хранилища:

| Что замокано | Где находится                                                                                                                                                                                                                                                                                     |
|--------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Поиск лекарств и карточка препарата** | Модуль `data` → пакет `com.example.data.repository` → класс **`DrugRepositoryImpl.kt`**. Внутри — список `mockDrugs` (лекарства с id, названием, производителем, ценой, описанием). Методы `searchDrugs()` и `getDrugById()` работают с этим списком. В моках реализовано 6 лекарств с названием:Ibuprofen, IbuHEXAL, IbuProLong, Paracetamol, Aspirin, Ibuklin |
| **Закладки и история поиска** | Тот же **`DrugRepositoryImpl.kt`**: методы `getBookmarks()`, `addBookmark()`, `removeBookmark()`, `getSearchHistory()`, `clearSearchHistory()` возвращают пустые списки или ничего не сохраняют (TODO под реальное хранилище).                                                                    |

Название: Поиск лекарства по названию

Пользователь: Анна, 42 года

Цель: Быстро найти информацию о лекарстве

Предусловия: Приложение установлено

Основной поток:
1. Открывает приложение
2. Вводит название препарата в строку поиска
3. Нажимает «Поиск»
4. Приложение отправляет запрос на сервер
5. Сервер возвращает список препаратов
6. Пользователь выбирает нужный препарат
7. Просматривает карточку лекарства


Альтернативный/ошибочный поток
1. Препарат не найден и приложение показывает «Нет результатов»
2. Пользователь проверяет написание или использует поиск по частичному названию, тоесть написать можно написать только начальную часть названия лекартсва
